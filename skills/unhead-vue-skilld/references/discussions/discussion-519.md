# Possible memory leak in XSS management

Hi, we are working on a Nuxt application that uses unhead 1.x under the hood to manage the behaviour of the useHead composables. What we are noticing in production is that the replace function used by the RegExp in the XSS plugin stays in memory creating a memory leak. I'm not saying that the leak is generated by the incorrect implementation of the plugin, at this point I blame RegExp implementation in Node (we're using 20.18 "slim" version). 
Does this sound familiar? Anybody here experienced the same issue? Did you find any possible workaround?
