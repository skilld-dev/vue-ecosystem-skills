# **Unhead v3: Streaming SSR & Sync Rendering**

Harlan Wilton

**4 min read**

Published March 10, 2026

I'm excited to announce that Unhead v3 is now in beta, the biggest architectural shift since v2 - bringing streaming SSR support, fully synchronous rendering, and significant performance gains.

## [TL;DR - What's New in v3](#tldr-whats-new-in-v3)

- **Streaming SSR**: Render head tags progressively as your page streams
- **Sync Rendering**: `renderSSRHead()` and `renderDOMHead()` are now synchronous
- **Pluggable Renderers**: New `head.render()` API with factory-based architecture
- **Improved Types**: Discriminated unions for `useHead()` with tag-level type narrowing
- **Performance**: ~32% faster SSR, ~11% smaller client bundle
- **Validate Plugin**: Catch SEO mistakes with typed, ESLint-style rule config and source tracing
- **Cleanup**: All v2 deprecations removed

## [Streaming SSR](#streaming-ssr)

The headline feature of v3 is streaming SSR support. Instead of waiting for your entire page to render before emitting head tags, Unhead can now stream them progressively as components render.

This is powered by a queue-based pattern where head entries are serialized as inline `<script>` tags that push data to `window.__unhead__`. A lightweight bootstrap script on the client consumes the queue before your main JS bundle loads.

```
import { createStreamableHead } from 'unhead/stream'

const head = createStreamableHead()
```

Streaming works with Vue, **React**, Solid.js, and **Svelte** - each framework integration provides streaming exports.

## [Synchronous Rendering](#synchronous-rendering)

All rendering in v3 is now synchronous. This was a prerequisite for streaming support but also simplifies the API surface considerably.

```
- const head = await renderSSRHead(head)
+ const head = renderSSRHead(head)

- await renderDOMHead(head, { document })
+ renderDOMHead(head, { document })
```

All hooks (`dom:beforeRender`, `ssr:beforeRender`, `tags:resolve`, etc.) must also be synchronous. Promise-based inputs are no longer supported.

## [Pluggable Renderers](#pluggable-renderers)

Rendering is now factory-based. `createUnhead()` accepts a renderer as its first argument, and exposes a unified `head.render()` method.

```
import { createUnhead } from 'unhead'
import { createServerRenderer } from 'unhead/server'

const head = createUnhead(createServerRenderer(), {
  // options
})

const result = head.render()
```

This replaces the direct `renderSSRHead()`/`renderDOMHead()` pattern (which still works but is deprecated) and enables better tree-shaking since resolve logic is only bundled when needed.

## [Improved Types](#improved-types)

`useHead()` now uses discriminated union types for much tighter type safety:

- **Link tags** narrow by `rel` - `CanonicalLink`, `StylesheetLink`, `PreloadLink`, etc.
- **Preload links** further narrow by `as` attribute
- **Script tags** narrow by `type`/`src` - `ExternalScript`, `JsonLdScript`, `InlineScript`
- **Meta tags** enforce mutual exclusion of `name`, `property`, `http-equiv`, and `charset`

This catches common mistakes at the type level rather than runtime.

## [Performance](#performance)

Building on v2's performance work, v3 delivers further improvements through a leaner core and smarter bundling.

**Bundle Size**

| **Build** | **Δ size** | **Δ gzip** |
| --- | --- | --- |
| Client | -10.9% | -11.2% |
| Server | -4.5% | -4.6% |
| Vue Client | -9.9% | -10.2% |
| Vue Server | -4.1% | -4.1% |

**Runtime**

| **Benchmark** | **Improvement** |
| --- | --- |
| @unhead/vue e2e | 32% faster |
| Core e2e | 17% faster |
| @unhead/vue simple | 26% faster |
| Core simple | 28% faster |

Key optimizations include client-only capo sorting (server already sorted, no need to re-sort on client), moving `resolveTags()` to a tree-shakeable standalone import, and removing hooks from the core to minimize the streaming IIFE.

## [Validate Plugin](#validate-plugin)

New optional `ValidatePlugin` catches common head tag mistakes that TypeScript can't - relative canonical URLs, missing OG companions, typos in meta properties, conflicting robots directives, and more. Rules are configured with an ESLint-style flat config, and each warning includes the source location of the `head.push()` call.

```
import { ValidatePlugin } from 'unhead/plugins'

const head = createHead({
  plugins: [
    ValidatePlugin({
      root: process.cwd(),
      rules: {
        'missing-description': 'off',
        'viewport-user-scalable': 'info',
      }
    })
  ]
})
// [unhead] og:image should be an absolute URL (./src/components/Hero.vue:18:3)
```

All 19 rule IDs are fully typed with autocomplete via `ValidationRuleId`. See the **Validate Plugin guide** for the full rules table.

## [Dropped Deprecations](#dropped-deprecations)

All v2 deprecations have been removed. If you're still using any of these, update before migrating:

| **Removed** | **Replacement** |
| --- | --- |
| `children` | `innerHTML` |
| `hid` / `vmid` | `key` |
| `body: true` | `tagPosition: 'bodyClose'` |
| `useServerHead` | `useHead` |
| `useServerSeoMeta` | `useSeoMeta` |
| `createHeadCore` | `createUnhead` |
| `headEntries()` | `head.entries` Map |
| `PluginSchemaOrg` | `UnheadSchemaOrg` |
| `Head` type | `HeadTag` |
| `MetaFlatInput` type | `MetaFlat` |

Hooks `init`, `dom:renderTag`, and `dom:rendered` have also been removed.
