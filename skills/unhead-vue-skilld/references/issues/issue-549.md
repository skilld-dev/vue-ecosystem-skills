# Vue 3 documentation makes unsupported statement

### Describe the change

Page states:


"While Vue does not recommend usage of the Options API, it is a valid way to build Vue 3 applications.".

However the Vue docs do not support this statement.  Quote from the official Vue 3 documentation:

The Options API "remains a solid choice for many low-to-medium-complexity scenarios"  in the "Introduction".
It then provides a comparison between the Options API and the Composition API, highlighting use cases and benefits, but does not recommend against using the Options API.

In answer to an FAQ about future support for the Options API it states:

"No, we do not have any plan to do so. Options API is an integral part of Vue and the reason many developers love it."
